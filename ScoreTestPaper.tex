\section{Introduction}
\label{sec:introduction}

Model selection is an important step for
statistical analysis, allowing the practitioner to confirm the
reliability of their chosen model and any resulting
conclusions. Well-known model selection methods include likelihood
ratio tests (LRTs), Akaike's Information Criterion (AIC), and score 
tests. Comparing two nested models via either LRTs or AIC means 
 using maximised log-likelihood values to assess the quality of
each model, while applying a penalty for model
complexity. For instance, conducting a LRT requires twice the difference of each
model's maximised log-likelihood. This is 
compared to a null distribution, where
the degrees of freedom can be thought of as containing the penalty for
model complexity \parencite{vaartAsymptoticStatistics2012}. 
Using AIC involves directly adding a penalty term to each
maximised log-likelihood and comparing the results \parencite{burnham2002}. 

To compare two nested models using a score test, a score test
statistic is calculated and compared to the same null
distribution as the LRT statistic. However, unlike the LRT, the score
test does not require both models to be fitted, but instead
employs gradients to inform the improvement in fit available by
perturbing the parameter estimates from the simpler model
\parencite{rao2005}. Specifically, the score test statistic is
calculated using the
score vector and information matrix for the more complicated model,
evaluated at the parameter estimates of the simpler model. See
\textcite{rao2005} for an overview and recent developments of score
tests. 

Each method has advantages and
disadvantages. It is straightforward to perform the
calculations required to compare models via LRTs or AIC. However, both
models must be fitted. To conduct score tests, only the simpler of the
two models needs to be fitted \parencite{rao2005}. However, the required
gradients can be challenging to obtain, often acting as a
deterrent for conducting these tests. 

Historically, LRTs and AIC have been the most widely-used methods
of model selection. However, when the models under comparison require
considerable time and resources to fit, these methods of comparison can
become inconvenient. This issue is amplified when considering
multiple potential models for a data set, as every model in the
candidate set must be fitted, potentially rendering comprehensive
model selection impractical.

In such cases, turning to score tests may be
beneficial, but for usability, we require a straightforward way of
calculating the gradients needed for the score test
statistic. Traditionally, options for obtaining these gradients
included symbolic differentiation and numeric approximations. The former
can quickly become unrealistic as model complexity grows, while the
latter often produces unstable and inaccurate
results \parencite{ADMBpaper}. However, a third option that has not so
far been widely used is automatic differentiation (AD). In this paper,
we demonstrate that recent AD tools developed for optimisation can be
repurposed to provide a newly straightforward implementation of score tests. 

By algorithmically harnessing the chain rule of differentiation, AD tools produce exact
gradients of any given function \parencite{ADMBpaper}. Using these
tools only requires the practitioner to supply the objective function 
\parencite{TMBpaper}. For statistical models, these tools are 
mainly used for model-fitting or calculating 
the Laplace approximation. We show that utilising the side-effects of these
capabilities -- repurposing their ability to calculate
exact log-likelihood gradients -- facilitates a
convenient implementation of score tests.

This has implications for a wide range of models. For instance, for
random effects models, intractable
likelihoods involving potentially high 
dimensional integrals produce gradients with no closed form, resulting in
a host of challenges. Model fitting previously required tailored
software, even for relatively simple examples such as
generalised linear mixed models (GLMMs), although gradients remained
elusive \parencite{lme4-pkg}. \textcite{wang2022} present extensive derivations
to obtain GLMM gradients, enabling previously
inaccessible tasks such as score tests.
However, there remain many random effects 
models for which such solutions and derivations don't exist. Random effects
models are just one illustration of the  
advancements offered by AD. 

To illustrate the benefits of score tests 
and AD tools, we will focus on the class of random effects models known as
spatial capture-recapture (SCR) models, used for estimating density
and distribution of animal populations. Model selection is often difficult 
for SCR models. Likelihood equations for these models involve products of two-dimensional spatial
integrals \parencite{borchersEfford2008}. This makes model-fitting, and therefore
using AIC to compare models, time consuming and resource intensive,
especially when many models are being considered. However, an absence
of suitable alternatives means comparison via AIC is the
primary method of SCR model selection
(e.g. \cite{secr-aic-ex}). Removing the barriers to conducting score
tests  presents a new and potentially impactful solution to this issue. 

We will explore the gains of conducting score tests using AD tools
via the framework introduced by \textcite{catchpoleMorgan1996}, and
later developed by \textcite{mccreaMorgan2011}. Following the
convention of organising the
candidate set into a tree structure, the framework provides an efficient way of
navigating through the set using score tests, 
amplifying the benefits of score tests for model selection. The issue
of choosing a model from a   
potentially vast candidate set of models of varying complexity is reduced to the
task of calculating score statistics and fitting a few, potentially
relatively simple, models. 

We present our findings using an SCR
data set of ovenbird captures \parencite{secr-pkg} with many potential
models to consider. We employ
the \lstinline{RTMB} R package \parencite{rtmb-pkg}, a widely-used AD tool, to
implement score tests for SCR model selection. Compared to traditional
methods, we examine how our approach enhances the 
convenience and accessibility of model selection, while accommodating
potentially common issues such as non-nested models.

Our proposed model selection approach is widely applicable, enabling
accessible model selection for many classes of models across a vast
range of fields.  

\section{Model selection using score tests}
\label{sec:model-sel}

\subsection{Calculating score test statistics}
\label{sec:calculating-score-stat}

We begin with a brief overview of score tests, refer to texts such
as \textcite{rao2005} for more detailed information. When
comparing two nested models 
using a score test, the null hypothesis 
states the simpler model is adequate. We will refer to this
 model as the `$H_0$ model', and the more complex model as the `$H_1$
model', with parameter vectors $\thetaZero$ and
$\thetaOne$, respectively. We define $\thetaOneTilde$ as the
$\thetaOne$ vector that shares the constraints of the $H_0$ model and
uses the estimates of $\thetaZeroMLE$. By definition, this vector
makes the $H_1$ model equivalent to the fitted $H_0$
model; see below for an example. Lastly, we refer to $\boldsymbol{u}_1
(\boldsymbol{\theta})$ as the gradient 
vector, also called the score, of the $H_1$ log-likelihood at
$\boldsymbol{\theta}$.

For a simple illustrative example to clarify $\thetaOneTilde$, consider $Y_i \sim
\text{Normal}(\mu_i, 1)$, where $H_0: \mu_i = \alpha$ and $H_1: \mu_i
= \alpha + \beta x_i$ for some covariate $\boldsymbol{x}$, with
maximum likelihood estimate (MLE) vectors $\thetaZeroMLE$ and
$\thetaOneMLE$, respectively. For this model, $\thetaZeroMLE =
\hat{\alpha}$, and therefore 
$\thetaOneTilde = (\hat{\alpha}, 0)$, as under the $H_0$
model, $\beta=0$.

Briefly, score tests are z-tests summarising how `surprising'
$\scoreU$ is under $H_0$. Under $H_0$, the
expectation of this gradient is 
$0$. That is, $E[\boldsymbol{u}_1
(\thetaOneTilde^*)]=0$, where $\thetaOneTilde^*$ is the true
vector under $H_0$, as shown in Section \ref{sec:exp-zero-proof} of
the Appendix. Using one dimension for illustration, the z-test
statistic is therefore $
\left.
  \{\scoreUOneDim - E[u_1(\thetaOneTildeOD^*)]\}
  \middle/
  \sqrt{\text{Var}\{\scoreUOneDim}\}
\right. =
\left.
  \scoreUOneDim \middle/ \sqrt{\text{Var}\{\scoreUOneDim}\}
\right.$. It can be shown that $\text{Var}\{\scoreUOneDim\} =
\mathcal{I}(\thetaOneTildeOD)$ -- refer to Section
\ref{sec:var-info-equiv-proof} of the Appendix for the algebraic
justification. Therefore, the z-statistic can also be expressed as
$\left. \scoreUOneDim \middle/ \sqrt{\mathcal{I}(\thetaOneTildeOD)}
\right.$, taking into account the overall shape of the likelihood to
assess the magnitude of $\scoreUOneDim$. 

Figure \ref{fig:fig1a} highlights the necessity of standardising via a
z-statistic, as
the value of $\scoreUOneDim$ alone is insufficient. In this
figure, we consider $Y_i \sim \text{Normal}(\mu, 1)$, where
$H_0: \mu = 0$ and $H_1: \mu = \alpha$, and both log-likelihood curves
share $\scoreUOneDim=5$. However, as 
confirmed by the LRT statistics, the $H_0$ model is adequate for the
data of the orange curve, yet inadequate for the data of the blue
curve.

As $\scoreUOneDim$ is a sum of independent contributions, the
z-statistic is asymptotically Normal$(0,1)$ under $H_0$, as is
customary. Therefore, a two-tailed 
p-value may be obtained to assess $H_0$. However, following convention, the score test
statistic is found by squaring this standardised value,
i.e. $S=\frac{\scoreUOneDim^2}{\mathcal{I}(\thetaOneTildeOD)}$,
resulting in 
an asymptotic Chi-squared distribution.

The above reasoning holds as model dimension increases. Therefore, in general,
$S = \scoreU^T \scoreI^{-1}
\scoreU$, where $\scoreU$ is the score vector of the $H_1$ model
evaluated at $\thetaOneTilde$ and $\scoreI$ is the information
matrix. In calculating
$S$, we will focus on using the observed information matrix. See the
Discussion for more details.

% -- as in z-test, rely on asymptotic Normality to produce a p-value
% assessing model fit 
% -- standardise = score/sqrt(information)
% -- and by the nature of the score (sum of independent contributions to
% log-likelihood), this is asymptotically Normal
% -- squaring means that the test becomes two-sided. Also gives same
% chi-sq distribution as LRT, for example

As shown in one dimension in Figure \ref{fig:fig1b}, $S$ is also obtainable
as an approximation-based LRT statistic based on a quadratic approximation to the
log-likelihood. This is derived in one dimension in Section \ref{sec:lrt-s-equiv-proof} of
the Appendix. Therefore, when the log-likelihood is quadratic, the LRT
statistic and score statistic are exactly equal, and otherwise they are
asymptotically equal. The null distribution for both score tests and
LRTs is a Chi-squared
distribution where the degrees of freedom is the difference in the
number of parameters for the $H_0$ and $H_1$ model.

Previously, the obstacle in implementing score tests was obtaining the
score vector, $\boldsymbol{u}_1 (\thetaOne)$, and information
matrix, $\mathcal{I}_1 (\thetaOne)$, to evaluate at
$\thetaOneTilde$. Using recent software advancements, we employ 
the \lstinline{RTMB} R package -- a commonly-used AD tool -- to do
this. Translating the $H_1$ log-likelihood into 
an R function, it becomes straightforward to extract the score vector
and information matrix. Using AD to also fit
the $H_0$ model can provide time savings, especially for sophisticated models.

An illustration of using \lstinline{RTMB} and symbolic differentiation
is given in Section \ref{sec:score-test-rtmb-ex}, where $S$ is found
for the example with $Y_i \sim 
\text{Normal}(\mu_i, 1)$, where $H_0: \mu_i = \alpha$ and $H_1: \mu_i
= \alpha + \beta x_i$ for some covariate $\boldsymbol{x}$. We
emphasise that although these models are particularly simple,
obtaining $\scoreU$ and $\scoreI$ is surprisingly
involved. Furthermore, the symbolic
differentiation required to calculate $S$ increases considerably as
model complexity increases, typically becoming impractical.

\subsection{Implementing the score test framework}
\label{sec:implementing-framework}

By organising the candidate set into a tree structure, the model
selection framework of \textcite{catchpoleMorgan1996}
defines an efficient path to the final selected
model, beginning by considering the simplest models. This is done
through the construction of `levels', and the use of score tests to
select the best model at each level.

As described in \textcite{mccreaMorgan2011}, 
the most basic model defines Level 0 of the candidate set, and Level 1
contains all models 
with one added parameter dependence. For instance, in the example
introduced in Section \ref{sec:calculating-score-stat} with $Y_i \sim
\text{Normal}(\mu_i, 1)$, where $H_0: \mu_i = \alpha$ and $H_1: \mu_i
= \alpha + \beta x_i$, if $H_0$ is the Level 0 model, then $H_1$ is a
Level 1 model. Score tests are used to compare all Level 1 
models to the Level 0 model. As in 
Section \ref{sec:calculating-score-stat}, we refer to the Level 0
model as the `$H_0$ model', and the Level 1 models as `$H_1$
models'. The $H_1$ model associated with the smallest significant
p-value defines the new null hypothesis -- that is, it becomes the new
$H_0$ model. Level 2 is then constructed, consisting of all models with one
additional parameter dependence relative to this model, and the
process is repeated. We stop at the level where all of the score
tests produce p-values that are large enough for the analyst to be
satisfied with the $H_0$ model. At this point, the $H_0$ model becomes the
chosen model.  

Due to the level structure, implementing this
framework on a comprehensive candidate set typically only requires a
few models be fitted, as only the $H_0$ models need to be fitted for
the score tests. Furthermore, the stepwise nature of the  
framework promotes convenient model-fitting -- the MLEs
from a given $H_0$ model may provide appropriate starting values for
the next $H_0$ model \parencite{mccreaMorgan2011}. Combining these
benefits with the convenience of AD, using the framework of
\textcite{catchpoleMorgan1996} requires fewer resources and less time
than traditional model selection techniques, especially for a
comprehensive candidate set. Fitting all models under consideration is
replaced by accessible calculations and fitting a few models. 

\subsection{Non-nested models, boundary considerations and unidentifiability}
\label{sec:non-nested-and-boundary-consid}

When models are non-nested, it is impossible 
to obtain the $H_0$ model from the $H_1$ model. No
$\thetaOneTilde$ vector exists, so a
score test statistic cannot be calculated.

Issues also occur when the regularity conditions underlying score
tests do not hold. Most commonly, problems arise when values in
$\thetaOneTilde$ lie on the boundary of the parameter space or are
unidentifiable under the $H_0$ model. The null Chi-squared distribution no
longer holds, so the null distribution must be obtained symbolically
or via simulation. For example, texts such as
\textcite{andrews-2001} symbolically obtain null distributions for
selected models. 

Specifically, unidentifiability can cause identically zero and
therefore uninformative score statistics. For
further information, refer to texts such as
\textcite{cox-hinkley-1979} and \textcite{lee-chesher-1986}. Briefly,
identically zero statistics arise when all parameters that 
distinguish the $H_1$ and $H_0$ model are unidentifiable. For
example, consider a $H_0$ non-mixture model with
likelihood $f(\boldsymbol{y}|\theta)$, and a $H_1$ two-component mixture model
with likelihood $(1-\pi)f(\boldsymbol{y}|\theta_A) +
\pi f(\boldsymbol{y}|\theta_B)$. Under
$H_1$, observations can fall into group A or B, with probability
$(1-\pi)$ and $\pi$ respectively, and under $H_0$ all observations
are in the same group. Therefore, parameters $\pi$ and $\theta_B$
distinguish both models and
are unidentifiable under $H_0$. No matter how $\thetaOneTilde$ is constructed to
align with $H_0$ constraints (e.g. setting $\theta_A=\theta_B$ or $\pi=0$),
$\scoreU$ in the direction of these parameters will
always be $0$. As the score test `depends' on these derivatives, the
score statistic is identically zero. 

When such issues arise, we propose separating the
candidate set into groups -- where models in each group fall into a
nested structure with no boundary or identifiability problems -- and carrying out the
framework of \textcite{catchpoleMorgan1996} on each group, using AIC to
make a final selection from the chosen models. Using AIC like this
is unlikely to require any extra models be fitted. See Section
\ref{sec:ovenbirds} for an example. 

\section{Application: ovenbirds}
\label{sec:ovenbirds}

\subsection{Data set}
\label{sec:data-set}

In \textcite{borchersEfford2008},
the authors illustrate SCR model selection using AIC. The data
set used consists of captures of birds 
(red-eyed vireos) from a mist-netting survey on the Patuxent Research Refuge 
in Maryland, USA, conducted by C.S. Robbins from 1961 to 1972. Considering
a range of parameter dependencies, a candidate set of 
fifteen potential models is considered, and the model with the lowest
AIC value is selected as the final model.

Working with a similar data set allows consideration of the same
parameter dependencies, and comparison to a well-known
example. The `ovenbird' data set, 
available via the `secr' R package, contains adult captures from a 
mist-netting survey of ovenbirds conducted from 2005 to 2009 at the Patuxent
Research Refuge by D. K. Dawson and M. G. Efford. As
described in the package documentation, the forest in the area has
changed little since the previous survey conducted by C.S. Robbins
\parencite{secr-pkg}. As we are only illustrating model selection, not
drawing model-based inference, it seems reasonable to consider the 
same parameter dependencies as 
\textcite{borchersEfford2008}. By considering all possible
combinations of these dependencies, we construct a comprehensive
candidate set of forty five models. 

Forty four mist nets were passively operated for nine or
ten non-consecutive days in late May and June of each year. One bird
was killed by a predator while in a net 
in 2009 \parencite{secr-pkg}, and for our analysis we remove this loss
from the data set. We retain any 
records of individual birds being captured more than once in a
day. 

To construct an accurate picture of the benefits afforded by model
selection via score tests and AD tools, alongside our proposed
approach, we implement model selection via
AIC. Using an extensive candidate
set, we conduct a comprehensive comparison of a novel, more efficient,
method of model selection with a traditional approach. 

\subsection{Implementing model selection}
\label{sec:implementing-for-ex}

Evaluating an SCR likelihood requires a product of 
two-dimensional integrals, where each integral is an infinite sum over
the model's random effects \parencite{borchersEfford2008}. Therefore,
fitting every model in a comprehensive candidate set 
such as ours requires significant resources and time. Despite this,
the convenience of calculating AIC means they are often used for
SCR model selection \parencite{efford-schofield}. Although popular models
for estimating animal abundance, the complexity of SCR likelihoods
meant derivatives and therefore tools such as score tests were
previously elusive. The benefits of simplifying model selection via
our proposed approach are substantial, allowing newly accessible
model selection for these models.  

Please refer to Section \ref{sec:overview-candidate-models} of the Appendix
for an overview of the parametrisation of the SCR models in our
candidate set. Briefly, each model estimates three parameters -- $D$,
$g_0$ and $\sigma$ -- and we consider a range of possible dependencies
for these parameters. By considering all combinations of these dependencies, we 
have forty five models in our candidate set. See \textbf{Tables (no)} in
the Results for a complete list of models.

Due to the comprehensive nature of the candidate set, we encounter
unidentifiability as described in Section
\ref{sec:non-nested-and-boundary-consid}, potentially resulting in identically
zero score statistics. Specifically, in the
candidate set, we consider models with a two-component finite mixture
for $g_0$ and/or $\sigma$. Unidentifiability issues arise when
comparing models that don't share these mixture components, as in the
example of Section \ref{sec:non-nested-and-boundary-consid}. To resolve this, we
only conduct score tests comparing models with the same mixture
dependencies for $g_0$ and $\sigma$. Therefore, we construct four
groups for model selection: one with no mixture dependencies, one with
mixture dependencies for $g_0$ only, one with mixture dependencies for
$\sigma$ only, and one with mixture dependencies for $g_0$ and
$\sigma$. If we were also considering models with different detection
functions -- i.e. non-nested models, as described in Section
\ref{sec:non-nested-and-boundary-consid} -- we would also separate
models by detection function. 

To conduct model selection, we define the objective function
(the negative log-likelihood) for every candidate model as an R 
function. For each group, we carry out the framework of
\textcite{catchpoleMorgan1996} to navigate through the candidate
models. We identify the Level 0 model and, using 
\lstinline{RTMB} as outlined in Section \ref{sec:rtmb-appendix}  
of the Appendix, we efficiently conduct score tests comparing the
Level 0 model to all Level 1 models. We continue with the score test
framework as described in Section \ref{sec:implementing-framework},
stopping at the level where p-values are no longer significant. Using
AIC, we compare the final model selected for each group,
selecting the model with the smallest AIC as the final model. 

To illustrate the benefits of this approach, we also implement model selection as
it would likely currently be done in practice. We fit every model using the
\lstinline{secr.fit} function from the \lstinline{secr} R package, and
compare them using AIC, calculated using the \lstinline{AIC}
function from the same package.  

\subsection{Results}
\label{sec:ex-results}

Implementing model selection using the approach outlined in Section
\ref{sec:implementing-for-ex}, harnessing score tests via \lstinline{RTMB}

implementing model selection via AIC takes approximately 24.8

\section{Discussion}
\label{sec:discussion}

\subsection{General considerations for the score test framework}
\label{sec:disc-general}

Generally, when performing model selection, it is necessary to take model
uncertainty into account, e.g. via a bootstrap simulation, so standard
errors are not underestimated \textbf{(Claeskens and Hart
  2008?)}. Furthermore, texts such as \textcite{mccreaMorgan2011} 
recommend using simulation to select an appropriate conservative
level, mitigating against an inflated false positive rate due to multiple
tests. Refer to the text for more details. Lastly, with hierarchical
model selection it is generally 
possible to lose important model features. Therefore,
\textcite{mccreaMorgan2011} suggest the incorporation of step-down
tests, such as LRTs, in conjunction with score tests for model
selection.

\subsection{Applicability}
\label{sec:disc-applicability}

Our proposed model selection approach can be
applied to a wide range of models, the only requirements are those
underlying score tests. Therefore, applicability only requires
standard regularity assumptions, similar to those   
underlying LRTs. However, unlike LRTs, the local nature of score tests
requires these assumptions only hold in a neighbourhood around
$\thetaOneTilde$, rather than globally across the $H_1$ parameter
space. For example, in Section
\ref{sec:non-nested-and-boundary-consid} we only consider boundary
estimates and unidentifiability at $\thetaOneTilde$. The applicability
of our approach is therefore similar to, if not wider than, that of the
commonly-used LRT. 

\subsection{Observed vs expected information matrix}
\label{sec:disc-obs-exp-mat}

It is possible to use the observed or expected information matrix to
calculate score statistics, both have the same asymptotic
distribution \parencite{catchpoleMorgan1996}. Due to the considerable
gain in convenience, we generally recommend using the observed
information matrix. As illustrated in Section \ref{sec:rtmb-appendix},
with our approach this requires only a few lines of code that are
quick to run. In contrast, the expected information matrix requires
potentially extensive symbolic calculations, if tractable. For
sophisticated models, intractability often means numerical integration or
Monte Carlo simulation is needed to obtain an estimate of the expected
information matrix. As mentioned in Section \ref{sec:introduction}, the former
can be unrealistic as model complexity grows, while the latter can
require extensive time and resources.

Texts such as \textcite{catchpoleMorgan1996} and
\textcite{mccreaMorgan2011} explore the benefits afforded by the
expected information matrix. However, with the advancement of AD, these
benefits are now outweighed by the convenience of the observed
information matrix. \textcite{catchpoleMorgan1996} report closer  
alignment between score and LRT statistics with the expected information
matrix. As shown in Section \ref{sec:ex-results}, this does not
exclude close alignment when using the observed information  
matrix. Using simulation, \textcite{mccreaMorgan2011} find improvement in 
model selection accuracy from the expected information
matrix. The reported effect is not large and decreases as sample size
increases. Furthermore, observed information score statistics found
using AD over numeric approximations are more accurate, potentially
reducing these benefits.  

% In \textcite{mccreaMorgan2011}, the authors mention that a
% by-product of using the expected information matrix is detecting
% unidentifiability. While this is likely useful for large candidate
% sets of sophisticated models, the presence of alternative (likely more
% convenient) methods for detecting unidentifiability
% means this is insufficient to recommend the expected
% information matrix.  *** Check with Ben and Rachel -- OK to
% exclude?? 
 
A scenario where we recommend considering the expected information
matrix is when negative score statistics arise. As
explored in \textcite{morgan2007}, the observed information
matrix may occasionally result in a negative statistic, as a poor
fit under $H_0$ produces an observed information matrix that is not
positive definite. In this case, alternatives include
using the expected information matrix or the LRT
statistic. Specifically, when dealing with sophisticated models where
the required calculations are intractable
(e.g. SCR models), we recommend Monte Carlo methods to estimate
the former. For our purposes, this
means simulating data under the fitted $H_0$ model, evaluating the
$H_1$ observed information matrix at the $H_0$ MLEs for each data set,
and then averaging across all matrices. However, due to the likely loss in
efficiency, using the LRT statistic may be more
convenient. The asymptotic equivalence of score and LRT
statistics means accuracy is likely to be preserved.

\subsection{Concluding remarks}
\label{sec:disc-conclusion}

Harnessing recent software advancements with AD and the local nature
of score tests, we present an approach that considerably improves the
convenience of comprehensive model selection,
applicable to a wide range of statistical models. Through the lens of
the framework of \textcite{catchpoleMorgan1996}, only a small subset
of the candidate set must be fitted, further enhancing the time
savings. 

Further work includes simulation to confirm the absence of material
benefit when using the expected instead of the observed information
matrix, and further investigating the implementation of the considerations
in Section \ref{sec:disc-general}. 

\section{Appendix}
\label{sec:appendix}

\subsection{Score test proofs}
\label{sec:score-test-proofs}

For ease of explanation, in the following sections we work with
a one-dimensional $H_1$ parameter vector, $\thetaOneOD$, and a data set
consisting of one observation, $Y$. Therefore, $\llSimple$
represents the $H_1$ log-likelihood. We assume the required
regularity conditions for score tests hold. 

\subsubsection{$E[\boldsymbol{u}_1(\thetaOneTilde^*)]=0$}
\label{sec:exp-zero-proof}

The $H_1$ log-likelihood can be expressed as: $\llSimple
= \log \lrbr{\pdfSimple}$. Therefore, the score vector is:
$\scoreSimpleCond = \deltaThetaOneOD \log \lrbr{\pdfSimple} =
\frac{1}{\pdfSimple} \deltaThetaOneOD \lrbr{\pdfSimple}$.

Then, if $\thetaOneTildeODTrue$ represents the true parameter vector under
$H_0$, the expectation of $\scoreSimpleTrue$ -- an expectation
across data sets -- is given by:
\begin{equation}
  \begin{split}
    E_Y\lrsq{\scoreSimpleTrueCond} & = \intSimple \left. \lrsq{\frac{1}{\pdfSimple}
        \deltaThetaOneOD \lrbr{\pdfSimple}} \right|_{\thetaOneTildeODTrue}
    \pdfSimpleTrue dy \\ 
    & = \intSimple \left. \deltaThetaOneOD \lrbr{\pdfSimple}
    \right|_{\thetaOneTildeODTrue} dy = \left. \deltaThetaOneOD \lrbr{\intSimple
        \pdfSimple dy} \right|_{\thetaOneTildeODTrue} \\
  \end{split}
\end{equation}
as under the assumed regularity conditions, the order of
differentiation and integration can be swapped. 

And since $\intSimple \pdfSimple dy = 1$, we can therefore say
$E_Y[\scoreSimpleTrueCond]=0$ -- that is,
$E[\scoreSimpleTrue]=0$. Similar reasoning holds as the dimension of
the parameter space and data set increases, so in general,
$E[\boldsymbol{u}_1(\thetaOneTilde^*)]=0$. 

\subsubsection{$\text{Var}\lrbr{\scoreU} = \mathcal{I}(\thetaOneTilde)$}
\label{sec:var-info-equiv-proof}

Below, we will
use $\scoreSimple$ and $\scoreSimpleCond$ interchangeably, for
brevity.

Using reasoning similar to Section \ref{sec:exp-zero-proof}, it can be
shown $E_Y [\scoreSimple] = 0$, in general. Therefore,
$\text{Var}\lrbr{\scoreSimple} = E_Y[\scoreSimple^2]$. We 
can derive an expression for $E_Y[\scoreSimple^2]$, such that 
$\text{Var}\lrbr{\scoreSimple} = \mathcal{I}(\thetaOneOD)$.

Firstly, recall from Section \ref{sec:exp-zero-proof}:
$E_Y[\scoreSimple] = \intSimple \llSimpleFirstDeriv \pdfSimple
dy$. Therefore:
\begin{equation}
  \label{eq:exp-deriv}
  \begin{split}
  \deltaThetaOneOD E_Y[\scoreSimple] & = \intSimple \deltaThetaOneOD \llSimpleFirstDeriv \pdfSimple
  dy \\
  & = \intSimple \llSimpleSecondDeriv \pdfSimple + \lrbr{\llSimpleFirstDeriv^2}
  \pdfSimple dy \\
  & = E_Y[\llSimpleSecondDeriv] + E_Y[\scoreSimpleCond^2]
  \end{split}
\end{equation}
by the product rule of differentiation, and using the fact
$\llSimpleFirstDeriv = \frac{f'(y;\thetaOneOD)}{\pdfSimple} \implies
f'(y;\thetaOneOD) = \llSimpleFirstDeriv \pdfSimple$. As in
Section \ref{sec:exp-zero-proof}, standard regularity assumptions
allow swapping the order of differentiation and integration.

As $E_Y [\scoreSimple] = 0 \implies \deltaThetaOneOD
E_Y [\scoreSimple] = 0$, we therefore have:
\begin{equation}
  E_Y[\llSimpleSecondDeriv] + E_Y[\scoreSimpleCond^2] = 0
  \iff E_Y[\scoreSimple^2] = -E_Y[\llSimpleSecondDeriv]
\end{equation}

That is, $\text{Var}\lrbr{\scoreSimple} = \mathcal{I}(\thetaOneOD)$,
by definition of the Fisher information. Once again, similar
reasoning holds as model and data dimension increases -- thus,
generally, $\text{Var}\lrbr{\boldsymbol{u}_1 (\thetaOne)} =
\mathcal{I}(\thetaOne)$. 

\subsubsection{LRT and S equivalence}
\label{sec:lrt-s-equiv-proof}

Below, we use $\ell(\thetaOneOD)$ in place of $\llSimple$ for
brevity. We also use $\ellT$ to represent a second-order Taylor series
expansion of $\ell(\thetaOneOD)$, centered at
$\thetaOneTildeOD$. (Recall, $\thetaOneTildeOD$ is the parameter vector
specified under $H_0$, as described in Section
\ref{sec:calculating-score-stat}.) If $\thetaHatStar$ represents the
peak of this quadratic approximation, the approximation-based LRT
statistic comparing the $H_0$ and $H_1$ models is:
$2\lrbr{\ell(\thetaHatStar) - \ell(\thetaOneTildeOD)}$. 

And as seen in Section \ref{sec:calculating-score-stat}, when
$\thetaOneOD$ is one-dimensional, the score statistic is given by:
$\frac{\scoreUOneDim^2}{\mathcal{I}(\thetaOneTildeOD)}$. We will show
these two expressions are equivalent.

The aforementioned second-order Taylor series expansion is
given by: 
\begin{equation}
  \label{eq:taylor-exp}
  \ellT = \ellTilde + \ellTildeFirstDeriv(\thetaOneOD -
  \thetaOneTildeOD) +  \frac{1}{2} \ellTildeSecondDeriv(\thetaOneOD -
  \thetaOneTildeOD)^2 
\end{equation}

The gradient of this parabola is $\ellT' = \ellTildeFirstDeriv +
\ellTildeSecondDeriv(\thetaOneOD - \thetaOneTildeOD)$ (recall under a
second-order Taylor series expansion, derivatives with
order greater than $2$ are negligible). As $\thetaHatStar$
maximises the approximation (i.e. sets the gradient to $0$),
\begin{equation}
  \label{eq:theta-hat-star}
  \thetaHatStar = \thetaOneTildeOD -
  \frac{\ellTildeFirstDeriv}{\ellTildeSecondDeriv}
\end{equation}

Substituting Equation \ref{eq:theta-hat-star} into
Equation \ref{eq:taylor-exp}, 
\begin{equation}
  \ell_1(\thetaHatStar) =  \ellTilde -
  \frac{\ellTildeFirstDeriv^2}{2\ellTildeSecondDeriv} \iff
  2\lrbr{\ell(\thetaHatStar) - \ell(\thetaOneTildeOD)} =
  \frac{\ellTildeFirstDeriv^2}{-\ellTildeSecondDeriv}  
\end{equation}

That is, $2\lrbr{\ell(\thetaHatStar) - \ell(\thetaOneTildeOD)} =
\frac{\scoreUOneDim^2}{\mathcal{I}(\thetaOneTildeOD)}$, by definition
of the score vector and Fisher information. Once again,
similar reasoning holds as model and data dimension increase. 

\subsection{Score test example}
\label{sec:score-test-rtmb-ex}

\subsubsection{RTMB implementation}
\label{sec:rtmb-appendix}

%% Code is from `RCode.R' in 'Score-test-paper' folder :) -- just note
%% that in code here, have replaced 'beta0' with 'alpha' and 'beta1'
%% with 'beta'

We use the simple linear regression example from Section
\ref{sec:calculating-score-stat}, where $Y_i \sim \text{Normal}(\mu_i,
1)$, with $H_0: \mu_i = \alpha$ and $H_1: \mu_i =
\alpha + \beta x_i$. Using AD to fit the $H_0$ model in this
simple example does not necessarily provide a time saving, but is done
for illustrative purposes. Apart from the objective function and
its corresponding parameters, the code below generally applies
to calculating score statistics for different models.  

First, we define the objective
function (the negative log-likelihood) for the $H_0$ and $H_1$
model. 

\begin{lstlisting}[language=R]
nll <- function(pars) {
-sum(dnorm(y, pars$alpha + pars$beta*x, 1, log = TRUE))
}
\end{lstlisting}

Then, we create \lstinline{RTMB} objects for the $H_0$ and $H_1$ models. These
 contain exact derivatives, amongst other information on the
objective functions. 

\begin{lstlisting}[language=R]
obj.null <- MakeADFun(nll, list(alpha = alpha.start, beta = 0), map =
list(beta = factor(NA))) 
obj.alt <- MakeADFun(nll, list(alpha = alpha.start, beta = beta.start))
\end{lstlisting}

Next, we obtain the $H_0$ MLEs and define $\thetaOneTilde$. 

\begin{lstlisting}[language=R]
fit.null <- nlminb(obj.null$par, obj.null$fn, obj.null$gr)
pars.null <- list(alpha = fit.null$par["alpha"], beta = 0)
\end{lstlisting}

In place of symbolic differentiation, the following two simple lines of code
calculate $\scoreU$ and $\scoreI^{-1}$. 

\begin{lstlisting}[language=R]
score.null <- -obj.alt$gr(pars.null)
info.null <- obj.alt$he(pars.null)
\end{lstlisting}

And lastly, we obtain the score statistic and its p-value. 

\begin{lstlisting}[language=R]
S <- score.null %*% solve(info.null) %*% t(score.null)
p.val <- 1 - pchisq(S, df=1)
\end{lstlisting}

\subsubsection{Symbolic differentiation}
\label{sec:symb-diff-appendix}

To find $S$ via symbol differentiation, the few lines of code from above
are replaced by the calculations below. As mentioned in Section
\ref{sec:calculating-score-stat}, this example is simple. In reality,
similar calculations are likely to be much more complicated, if not
impossible. 

Recall that $Y_i \sim \text{Normal}(\mu_i,
1)$, with $H_0: \mu_i = \alpha$ and $H_1: \mu_i =
\alpha + \beta x_i$. We define the corresponding parameter
vectors as $\thetaZero = \lr{\betaZero}$ and $\thetaOne = \lr{\betaZero,
  \betaOne}$, respectively. We also denote the corresponding
log-likelihood values as $\ellZero$ and $\ellOne$.

To begin, we find the $H_0$ MLE. The pdf for one observed $Y_i$ value is
given by:

\begin{equation}
  f \left(y_i |  \thetaZero \right) = \frac{1}{\sqrt{2 \pi}} \exp
  \lrbr{\frac{-(y_i - \mu_i)^2}{2}}
\end{equation}

Therefore, the log-likelihood under $H_0$ is given by: 

\begin{equation}
  \ellZero \left(\thetaZero | \yBold \right) = \sumN \log
  \lr{\frac{1}{\sqrt{2 \pi}}} - \frac{(y_i - \betaZero)^2}{2} = \sumN
  - \frac{1}{2} \log(2 \pi) - \frac{(y_i - \betaZero)^2}{2}
\end{equation}

\begin{equation}
  \implies \frac{\delta \ellZero}{\delta \alpha} = \sumN -
  \frac{\delta}{\delta \betaZero} \lrbr{\frac{(y_i - \betaZero)^2}{2}}
  = \sumN - \frac{2 (y_i - \betaZero) (-1)}{2} = \sumN (y_i - \betaZero)
\end{equation}

And so,

\begin{equation*}
  \begin{split}
  \frac{\delta \ellZero}{\delta \betaZero} = 0 & \iff \sumN
  (y_i - \betaZeroMLE) = 0 \\
  \implies \frac{\delta \ellZero}{\delta \betaZero} = 0 & \iff \sumN y_i = n \betaZeroMLE
  \end{split}
\end{equation*}

\begin{equation}
  \implies \betaZeroMLE = \frac{\sumN y_i}{n} = \yBar
\end{equation}

Next, we find the score vector and observed information matrix of the
$H_1$ log-likelihood, evaluated 
at $\thetaOneTilde$. As mentioned in Section
\ref{sec:calculating-score-stat}, $\thetaOneTilde = \left(
  \hat{\alpha}, 0 \right) = (\yBar, 0)$. The $H_1$ log-likelihood is given by:
\begin{equation}
  \ellOne \left( \thetaOne | \yBold \right) = \sumN - \frac{1}{2} \log(2
  \pi) - \frac{(y_i - \betaZero - \betaOne x_i)^2}{2}
\end{equation}

And so,

\begin{equation}
  \frac{\delta \ellOne}{\delta \betaZero} = \sumN -
  \frac{\delta}{\delta \betaZero} \frac{(y_i - \betaZero - \betaOne
    x_i)^2}{2} = \sumN - \frac{2 (y_i - \betaZero - \betaOne x_i)
    (-1)}{2} = \sumN (y_i - \betaZero - \betaOne x_i)
\end{equation}

\begin{equation}
  \frac{\delta \ellOne}{\delta \betaOne} = \sumN -\frac{\delta}{\delta
  \betaOne} \frac{(y_i - \betaZero - \betaOne x_i)^2}{2} = \sumN -
\frac{2 (y_i - \betaZero - \betaOne x_i) (-1)}{2} = \sumN x_i (y_i -
\betaZero - \betaOne x_i) 
\end{equation}

That is,
\begin{equation*}
  \boldsymbol{u}_1 (\thetaOne) =
  \begin{bmatrix}
    \sumN(y_i - \betaZero - \betaOne x_i) \\
    \sumN x_i (y_i - \betaZero - \betaOne x_i)
  \end{bmatrix}
  \implies
  \boldsymbol{u}_1 (\thetaOneTilde) =
  \begin{bmatrix}
    \sumN(y_i - \yBar) \\
    \sumN x_i (y_i - \yBar)
  \end{bmatrix}
\end{equation*}

And since $\sumN (y_i - \yBar) = 0$, we can write:
\begin{equation}
  \boldsymbol{u}_1 (\thetaOneTilde) =
  \begin{bmatrix}
    0 \\
    \sumN x_i (y_i - \yBar)
  \end{bmatrix}
\end{equation}

To find the observed information matrix, we begin with the following
second partial derivatives.

\begin{equation}
  \frac{\delta^2 \ellOne}{\delta \betaZero^2} = \sumN \deltaBetaZero (y_i -
  \betaZero - \betaOne x_i) = \sumN (-1) = -n
\end{equation}

\begin{equation}
  \frac{\delta^2 \ellOne}{\delta \betaZero \delta \betaOne} =
  \sumN
  \deltaBetaOne (y_i - \betaZero - \betaOne x_i) = - \sumN x_i
\end{equation}

\begin{equation}
  \frac{\delta^2 \ellOne}{\delta \betaOne \delta \betaZero} =
  \frac{\delta^2 \ellOne}{\delta \betaZero \delta \betaOne} = - \sumN x_i
\end{equation}

\begin{equation}
  \frac{\delta^2 \ellOne}{\delta \betaOne^2} = \sumN \deltaBetaOne x_i (y_i -
  \betaZero - \betaOne x_i) = - \sumN x_i^2
\end{equation}

Therefore, the negative hessian is given by:
\begin{equation}
  - \frac{\delta^2 \ellOne}{\delta \thetaOne \delta \thetaOne^T} = -
  \begin{bmatrix}
    -n & - \sumN x_i \\
    - \sumN x_i & - \sumN x_i^2
  \end{bmatrix} =
  \begin{bmatrix}
    n & \sumN x_i \\
    \sumN x_i & \sumN x_i^2
  \end{bmatrix}
\end{equation}

As the negative hessian does not depend on $\thetaOne$, we have:  
\begin{equation}
  \mathcal{I}_1 (\thetaOneTilde) =
  \begin{bmatrix}
    n & \sumN x_i \\
    \sumN x_i & \sumN x_i^2
  \end{bmatrix}
\end{equation}

To calculate the score statistic, we will need the inverse of the
observed information matrix. 
\begin{equation}
  \scoreI^{-1} = \frac{1}{n \sumN x_i^2 -
    \lr{\sumN x_i}^2}
  \begin{bmatrix}
    \sumN x_i^2 & - \sumN x_i \\
    - \sumN x_i & n
  \end{bmatrix}
\end{equation}

Lastly, we calculate $S = \scoreU^T \scoreI^{-1} \scoreU$,
\begin{equation}
  S  = \frac{1}{n \sumN x_i^2 -
    \lr{\sumN x_i}^2} \lr{
  \begin{bmatrix}
    0 & \sumN x_i (y_i - \yBar)
  \end{bmatrix}
  \begin{bmatrix}
    \sumN x_i^2 & - \sumN x_i \\
    - \sumN x_i & n
  \end{bmatrix}
  \begin{bmatrix}
    0 \\
    \sumN x_i (y_i - \yBar)
  \end{bmatrix}
  }
\end{equation}

where:

\begin{equation*}
  \begin{bmatrix}
    0 & \sumN x_i (y_i - \yBar)
  \end{bmatrix}
  \begin{bmatrix}
    \sumN x_i^2 & - \sumN x_i \\
    - \sumN x_i & n
  \end{bmatrix} =
  \begin{bmatrix}
    -\sumN x_i \sumN x_i(y_i - \yBar) & n \sumN x_i (y_i - \yBar)
  \end{bmatrix}
\end{equation*}

Therefore: 

\begin{equation*}
  S = \frac{1}{n \sumN x_i^2 -
    \lr{\sumN x_i}^2} \lr{
    \begin{bmatrix}
      -\sumN x_i \sumN x_i(y_i - \yBar) & n \sumN x_i (y_i - \yBar)
    \end{bmatrix}
    \begin{bmatrix}
      0 \\
      \sumN x_i (y_i - \yBar)
    \end{bmatrix}
  }
\end{equation*}

\begin{equation}
  \implies S =
  \frac{n \lrsq{\sumN x_i (y_i - \yBar)}^2}{n \sumN x_i^2 - \lr{\sumN x_i}^2}
\end{equation}

$S$ can be compared to the $\chi^2_1$ distribution using an R function call
similar to the one in Section \ref{sec:rtmb-appendix}. 

\subsection{Brief  overview of SCR and the candidate set}
\label{sec:overview-candidate-models}

% $D$ describes animal density, while $g_0$ and $\sigma$ describe 
% the probability of being captured.

Using records of when and where individuals were captured, SCR models
allow for estimation of animal density and distribution
\parencite{borchersEfford2008}. As described in \textcite{secr-book},
there are two parts to an SCR model: the state model and the
observation model. The state model describes the distribution of the animal population,
the `biological reality we wish to describe'. The observation model
describes the sampling process we use to understand this
population. Selecting appropriate state and observation models to accurately
reflect the population and survey enhances the validity of
inference. For understanding, we briefly describe our selections below. Please refer to
texts such as \textcite{secr-book} for further details.

Specifying the state model means choosing a spatial point process to
describe the distribution of animals in the survey area
\parencite{borchersFewster2016}. It is common to select a spatial
Poisson process \parencite{secr-book}. Specifically, following the
example of \parencite{borchersEfford2008}, we use a
homogeneous Poisson process, so animals are assumed to be
distributed uniformly across the survey area. Therefore, in our candidate SCR models,
we estimate the parameter $D$, animal density in the area of interest.

To specify the observation process, we select a detection function for
our SCR models. This describes how an animal's probability of detection
changes as a function of the distance between their activity centre and a
detector, where an animal's activity centre is the centre of their home
range \parencite{secr-book}. The choice of detection function dictates
the `detection' parameters 
to be estimated. Although \textcite{borchersEfford2008} consider
SCR models with half-normal and hazard rate detection functions, we
only consider models with a 
half-normal detection function. Hazard rate detection
functions are seldom used and generally not recommended due to their
long tail \parencite{secr-pkg}. Therefore, our detection
parameters are $g_0$ and $\sigma$. 

In summary, our candidate SCR models estimate three parameters: $D$,
$g_0$ and $\sigma$. When fitting a SCR model, along with the
state/observation models, it is standard to specify any dependencies for $D$
and the detection parameters. Following the example set by
\textcite{borchersEfford2008}, we 
consider models with the dependencies shown in Table
\ref{table:dependencies}, where we specify the dependencies using the
parlance of the \lstinline{secr} R package. Considering all possible combinations of
these dependencies produces the forty five candidate models.

\begin{table}[h!]
  \centering
\begin{tabular} { |c|c|c|c|  }
  \hline
  \textbf{$D$ dependencies} & \textbf{$\sigma$ dependencies} &
                                             \multicolumn{2}{|c|}{\textbf{$g_0$
                                                               dependencies}} 
  \\
  \hline
  $D \sim 1$ & $\sigma \sim 1$  & $g_0 \sim 1$ & $g_0 \sim \text{b}$ \\
  \hline
  $D \sim \text{session}$ & $\sigma \sim \text{session}$ & $g_0 \sim
                                                           \text{session}$
             &  $g_0 \sim \text{session} + \text{b}$ \\
  \hline
  $D \sim \text{Session}$ & $\sigma \sim \text{h2}$ & $g_0 \sim
                                                      \text{h2}$ & \\
  \hline
\end{tabular}
\caption{All parameter dependencies considered in our candidate set.}
\label{table:dependencies}
\end{table}

In Table \ref{table:dependencies} (see \textcite{secr-book} for further detail):
\begin{itemize}
\item `$\sim 1$' represents a parameter being held pooled across
  sessions. For example, with $D \sim 1$, we have only one
  estimate of $D$ and use it for all sessions;
\item `$\sim \text{session}$' means a parameter has session-specific
  levels. If we specify $D \sim \text{session}$, we have a separate estimate
  of $D$ for each session;
\item `$\sim \text{Session}$' represents a trend across
  sessions. With `$D \sim \text{Session}$', we  have $D(y) =
  exp(\beta_0 + \beta_1 y)$, 
  where $y$ is the session number \parencite{borchersEfford2008}. In
  our case, $y=1,...,5$;
\item `$\sim \text{h2}$' denotes a
  two-class finite mixture model. For example, with `$\sigma \sim \text{h2}$', we assume
  there are two groups in the population: Group 1 uses $\sigma_1$,
  while Group 2 uses $\sigma_2$. An 
  individual falls in Group 1 with probability $p$ and in Group 2
  with probability $(1-p)$. These estimates are pooled across sessions; 
\item `$\sim \text{b}$' represents a trap response. With $g_0 \sim
  \text{b}$, we have two estimates of $g_0$: one for individuals
  captured on previously in a session, and one for individuals
  not yet been captured in a session. These estimates are pooled
  across sessions;
\item `$\sim \text{session} + \text{b}$' means we have
  session-specific levels and a trap response -- e.g. with $g_0 \sim \text{session} +
  \text{b}$, we fit a model with two separate estimates of $g_0$ for
  each session: one for previously captured individuals, and one for
  individuals not caught yet. 
\end{itemize}
