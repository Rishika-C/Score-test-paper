\section{Introduction}
\label{sec:introduction}

Model selection is an important step for
statistical analysis, allowing the practitioner to confirm the
reliability of their chosen model and any resulting
conclusions. Well-known model selection methods include comparing
models using likelihood ratio tests (LRTs), AIC values and score
tests. When comparing two nested models, each method has its own advantages and
disadvantages. It is straightforward to perform the
calculations required to compare models via LRTs or AIC values. However, both
models must be fitted. To conduct score tests, only the simpler of the
two models needs to be fitted. However, log-likelihood gradients from
the more complicated model are required to calculate the score test
statistic -- specifically, the score vector and information matrix of
the larger of the two models. This has often acted as a
deterrent for conducting these tests.

Historically, LRTs and AIC values have been widely-used methods
of model selection. Although, when the models under comparison require
considerable time and resources to fit, these methods of comparison can
become inconvenient. This issue is only amplified when considering
multiple potential models for a data set -- every single model in the
candidate set must be fitted, potentially rendering model selection
impractical, despite its value.

In such cases, turning to score tests would be
beneficial. Having to fit fewer models in the process of selecting a
final model for the data could only make model selection more
practical and accessible. Meaningfully contributing to this
practicality and accessibility requires a straightforward way of
calculating the gradients required by the score test
statistic. Traditionally, options for calculating these gradients
included manual calculations and numeric approximations. The former
can quickly become unrealistic as model complexity grows, while the
latter often produces unstable and inaccurate
results. In this paper, we propose the use of automatic
differentiation (AD) tools to solve this issue.

Employing the chain rule of differentiation, AD tools produce exact
gradients of any specified function. Using these tools often
only requires translating the desired function into an
appropriate coding language. For statistical models, these tools are
mainly used for optimisation 
(model-fitting), or calculating approximations such
as Laplace approximations. Utilising the side-effects of these
capabilities, by repurposing their ability to calculate
exact gradients, facilities a newly
convenient implementation of score tests.

This approach can greatly assist with model selection in fields such
as spatial capture-recapture (SCR). Model selection is often fraught
for SCR models. Likelihood equations for SCR models can be quite
sophisticated, involving products of two-dimensional spatial
integrals. Model-fitting, and therefore using AIC values to compare
models, is often resource-intensive,
especially when many models are being considered. Due 
to the difficulties in manually calculating or numerically 
approximating gradients, score tests have not generally been
used. Alternative approaches have been explored, but there are 
no well-established methods of model selection. Removing the barriers
to conducting score tests 
presents a new and potentially impactful solution to this issue. 

We will explore the gains of conducting score tests using AD tools
via the framework introduced by \textcite{catchpoleMorgan1996}, and
later developed by \textcite{mccreaMorgan2011}. Following the
convention of organising the
candidate set into a tree structure, the framework provides an efficient way of
navigating through the set using score tests, 
amplifying the benefits of score tests for model selection. The issue
of choosing a model from a   
potentially vast candidate set of models of varying complexity is reduced to the
task of calculating score statistics and fitting a few, potentially
relatively simple, models. 

We will use the `RTMB' R package -- a widely-used AD tool --
to implement score tests for model selection via this framework. We
present our findings using an illustrative example. We will use
the ovenbird SCR data set, available in the `secr' R
package. Considering a range of candidate models, we explore how
to extend the aforementioned framework to accommodate models that aren't
nested. Compared to traditional approaches, we examine how our
approach potentially enhances the convenience and accessibility of
model selection, in general. As with the possible difficulties of model
selection, our insights are not specific to any one field of
research. They may be applied to a vast range of models.

% into levels and, beginning with the simplest models, score tests
% define a path through these levels to select the
% final model \parencite{mccreaMorgan2011}. We begin by identifying the 
% most basic model -- this model 
% makes up Level 0 of the candidate set. Level 1 contains all models
% with one added parameter dependency. We begin with the null hypothesis
% that the Level 0 model is adequate for the data. Therefore, we refer to
% the Level 0 model as the `$H_0$ model'. Our
% alternative hypotheses are that each Level 1 model provides a
% significantly better fit, and we refer to these models as `$H_1$ models'. We conduct
% score tests comparing the null hypothesis to each alternative
% hypothesis, i.e. comparing the $H_0$ model to each $H_1$ model. The $H_1$ model for the 
% test with the smallest significant 
% p-value defines the new null hypothesis -- that is, it becomes the new
% $H_0$ model. Level 2 is then constructed,
% consisting of all models with one additional parameter dependency
% relative to this model, and the process is repeated. We stop
% when none of the score tests produce significant p-values. At this
% point, the $H_0$ model is the final chosen model.

% of the two models needs to be fitted \textbf{(cite)}. So at
